# Code of Conduct

HumanX contributors are expected to keep project spaces practical, respectful, and focused on the work.

## Expected Behavior

- Be direct and constructive.
- Assume good faith while still asking for evidence when technical claims matter.
- Keep criticism about code, design, documentation, or process.
- Respect maintainers' time by keeping issues and pull requests clear and reproducible.

## Unacceptable Behavior

- Harassment, threats, discriminatory language, or personal attacks.
- Publishing private information without permission.
- Repeated off-topic disruption.
- Deliberately submitting malicious code, secrets, or misleading security reports.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; block users; or report abuse to GitHub when needed.

Security reports should follow `SECURITY.md`, not public issue comments.
